## 3. 콘텐츠 규칙

### 구조
- 첫 3문단(intro) 안에 핵심 결론 제시
- 각 글은 특정 검색의도 1개를 명확히 해결
- 비교표/추천표/상황별 가이드 반드시 포함

### 데이터 기반 작성
- 식당 추천 시 반드시 실제 데이터(평점, 리뷰수, 가격) 근거 제시
- 리뷰 원문은 절대 인용 금지 — 키워드 기반 요약만 허용
- "맛있다", "유명하다" 등 근거 없는 주관적 표현 금지
- 데이터 기준일 명시 필수

### H2 규칙
- 사람이 읽는 소제목만 사용
- 금지: 날짜형, 불릿형, 이모지형, 잘린 문장형
- H2와 바로 다음 body 첫 문장이 동일하면 안 됨

### 내부 링크
- 앵커에 날짜·카테고리·블로그 잔재 문구 금지
- 설명형 앵커 사용 (예: "판교 고기 맛집 전체 보기")
- 같은 검색의도끼리 연결

## 4. 광고 규칙

| 조건 | 광고 상한 |
|---|---|
| 일반 포스트 (body ≤14) | 3개 |
| 장문 포스트 (body ≥15, h2 ≥6) | 4개 |
| 카테고리/지역 페이지 | 2개 |
| 절대 상한 | 4개 |

### 금지
- CTA 바로 앞뒤 광고 배치
- 이미지와 광고가 연속
- body 없이 ad가 연속

## 5. 이미지 규칙

| 글 유형 | 권장 이미지 수 |
|---|---|
| 지역 가이드 | 대표 1 + 본문 2 = 3장 |
| 카테고리 추천 | 대표 1 + 본문 1 = 2장 |
| 비교형 | 비교 인포그래픽 1 + 대표 1 = 2장 |
| 절대 상한 | 4장 |

### alt/caption
- alt: 구체적 설명 (나쁜: `맛집 이미지` / 좋은: `판교역 고기구이 맛집 가격 비교표 2026`)
- caption: 이미지가 왜 필요한지 설명

## 7. 금지 문구

### 본문에서 금지
- `많은 분들이 궁금해하시는`
- `한눈에 정리해드리겠습니다`
- `지금 바로 확인하세요` (CTA 밖에서 남발)
- 출처 없는 `TOP 5`, `BEST`, `맛집 순위`
- `업계 최고`, `압도적 1위`, `무조건 추천`
- 리뷰 원문 인용

### 구조 금지
- 제목만 바꾸고 결론이 같은 글
- 이미지 반복으로 문서 길이 늘리기
- 출처 없는 가격/평점 서술
- 광고 사이에 정보량이 없는 얇은 문단

## 8. 파일 규칙

### 신규 포스트 파일 표준
```javascript
const post = {
  id: [번호],
  sections: [
    { type: 'intro', html: '...' },
    { type: 'image', src: '/images/...', alt: '...', caption: '...' },
    { type: 'toc' },
    { type: 'ad', slot: '9463227631', format: 'autorelaxed' },
    { type: 'h2', id: '...', text: '...', gradientStyle: 'linear-gradient(...)' },
    { type: 'body', html: '...' },
    { type: 'ad', slot: '6297515693', format: 'auto' },
    { type: 'ending', html: '...' },
    { type: 'ad', slot: '6297515693', format: 'auto' }
  ]
}
export default post
```

### data/posts.js 메타데이터
```javascript
{
  "id": [번호],
  "slug": "[영문-키워드-slug]",
  "title": "[한국어 제목 | 부제]",
  "description": "[검색 스니펫용 설명 150자 이내]",
  "category": "[지역맛집|카테고리가이드|상황별추천|비교|시즌]",
  "region": "[samseong|jamsil|pangyo|yeongtong|mangpo|yeongtongGu|all]",
  "date": "YYYY-MM-DD",
  "tags": ["태그1", "태그2"],
  "thumbnail": "/images/post[ID]_thumb.svg"
}
```

### Slug 규칙
- 영문 소문자 + 하이픈 (한글/언더스코어 금지)
- 3~6단어, 핵심 키워드 포함
- 연도 포함 권장
- 예: `pangyo-lunch-meat-guide-2026`, `samsung-izakaya-best7-2026`

## 9. 커밋 규칙

```
[타입]: 설명
// 질문 의도: 한줄 요약
```

예시:
```
[crawl]: 네이버 크롤러 v1 작성 + 판교 데이터 수집
// 질문 의도: 판교 지역 네이버 식당 데이터 수집

[seo]: 카테고리 페이지 가이드 콘텐츠 800자 추가 (전 지역)
// 질문 의도: 카테고리 페이지 SEO 콘텐츠 강화

[post]: 신규 포스트 1~5번 작성 (판교 점심/회식/고기/일식/혼밥)
// 질문 의도: 포스팅 시스템 첫 콘텐츠 생성
```
