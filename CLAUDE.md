### 확장 스키마 (네이버 크롤링 후)

```js
{
  // 기존 유지 — 값 교체
  rt: 4.3,                    // Google → 네이버 평점
  cnt: 1520,                  // Google → 네이버 방문자 리뷰 수

  // 기존 유지 — 보강
  tags: [...],                // 네이버 리뷰 키워드 분석으로 자동 보강
  moods: [...],               // 리뷰 감성 분석으로 보강
  scene: [...],               // 리뷰 상황 분석으로 보강

  // rv — 내부 전용 (외부 노출 절대 금지)
  rv: [...],                  // 네이버 리뷰 원문 (검색엔진 키워드 분석 전용)
                              // 사이트에 리뷰 텍스트 직접 출력 금지
                              // AI 추천 reason 생성, 태그 추출에만 사용

  // 신규 추가
  naverPlaceId: "1234567890", // 네이버 Place ID
  naverBlogCnt: 342,          // 블로그 리뷰 수
  menuItems: [                // 대표 메뉴 + 가격
    { name: "된장찌개", price: 9000 },
  ],
  keywords: [...],            // 리뷰 키워드 분석 결과 (tags와 별도)
  naverUrl: "",               // 네이버 플레이스 직링크
  updatedAt: "2026-03-13",   // 마지막 갱신일
  parking: true,              // 주차 가능
  reservation: false,         // 예약 가능
}
```

### rv 필드 사용 원칙 (절대 규칙)

1. **rv 데이터는 사이트에 절대 직접 출력하지 않는다**
2. rv는 내부 검색엔진 고도화 전용:
   - AI 추천 시 reason 생성의 참고 자료
   - 키워드 추출 → tags, keywords, moods, scene 자동 매핑
   - 분위기·기분 태그 자동 생성
3. 사이트에서 리뷰를 보여줘야 하는 경우:
   - "방문자 키워드" 형태로 추상화해서 표시 (예: "가성비 좋음 · 국물 맛집 · 혼밥 가능")
   - 네이버 리뷰 원문 그대로 표시 금지

## 5. 핵심 운영 원칙

1. **사람이 "어디서 먹을지" 판단할 수 있게 써야 한다.** 글을 읽고 식당 선택이 가능해야 한다.
2. **검색엔진용 문장보다 사용자 판단 문장을 우선한다.**
3. **같은 템플릿을 복제하지 않는다.** 지역명만 바뀐 문단, 결론 반복 금지.
4. **리뷰 원문은 절대 외부에 노출하지 않는다.** 키워드 추출·태그 생성·AI 참고용 전용.
5. **데이터에 근거한 추천만 한다.** "맛집 TOP", "BEST" 등 근거 없는 표현 금지.
6. **기존 기능(AI추천, 룰렛, 카테고리)을 깨뜨리지 않는다.** 추가만 한다.
7. **광고는 콘텐츠를 보조해야지 콘텐츠를 해치면 안 된다.**

## 13. 콘텐츠 강화 대상

### A. 카테고리 페이지 (현재: 3줄 가이드 → 목표: 800~1,200자)

현재 하단에 짧은 가이드 2~3줄만 존재. 검색엔진이 유의미한 페이지로 판단하기엔 너무 얇음.

**추가할 내용:**

- 카테고리 소개 (이 카테고리의 특징, 가격대 범위, 평균 평점)
- 상황별 추천 (회식용 / 데이트용 / 혼밥용 / 점심용)
- 가격대별 가이드 (1만원대 / 2~3만원대 / 4만원+)
- 관련 카테고리 내부링크

### B. 지역 페이지 하단 (현재: 없음 → 목표: 1,000~1,500자)

현재 AI추천·전체목록·카테고리 탭만 존재. SEO 텍스트 없음.

**추가할 내용 (각 지역별 독창적):**

- "[지역명] 맛집 가이드 2026"
- 지역 맛집 문화, 카테고리별 추천 TOP3, 시간대별/가격대별 추천
- 주차·교통 팁, 계절별 추천

### C. 메인 페이지 하단 (현재: 5문단 → 목표: 8~10문단 + FAQ)

현재 SEO 텍스트가 있으나 부족. FAQ 스키마 없음.

**추가할 내용:**

- 서비스 소개 확장
- FAQ 스키마 3~5개
- 각 지역 특성 상세화
