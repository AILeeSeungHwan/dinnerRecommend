## 1. TeamCreate 선언문

포스팅을 새로 만들거나 기존 콘텐츠를 대량 수정할 때:

```text
포스팅 만들땐 TeamCreate로 에이전트 팀생성해줘.
에이전트 팀으로 프로젝트 코드를 병렬 리뷰하고, 문제를 수정하며,
에이전트 이용해서 계획승인모드로 각 팀원이 변경하기 전에 반드시 plan approval 받도록 해줘.
우리는 포스팅을 만드는 것이므로 각 팀원은 할당된 포스팅을 만들고 승인받은 뒤 생성하도록 해줘.
모든 팀원 모델은 Sonnet으로 설정해줘.
```

## 3. 작업 흐름

```
1. 주제 선정 & 검색의도 정의
   └─ Editor Lead가 주제를 배분하고 검색의도를 명시
2. plan approval 요청
   └─ Content Builder가 아래 내용을 포함한 계획서 제출:
      - 제목 (title)
      - slug
      - 검색의도 (1문장)
      - 대상 독자 (1~2문장)
      - 추천할 식당 목록 (data 파일에서 실존 확인)
      - H2 목록 (5~8개)
      - 이미지 계획
      - 광고 배치 계획 (3개 이하)
3. Editor Lead 승인
4. 초안 작성 (POST_TEMPLATE.md 준수)
5. 검수 (병렬)
   └─ SEO Reviewer: 검색의도/slug/제목/내부링크
   └─ Data Reviewer: 식당 실존여부/가격/평점 정확성/리뷰 미노출 확인
6. 수정 반영
7. Editor Lead 최종 승인
8. 파일 반영
   └─ posts/{id}.js 생성
   └─ data/posts.js 메타데이터 추가
   └─ sitemap 자동 갱신
9. 구문 검사 (node --check)
10. 커밋 & 배포
```

## 4. 포스팅 유형별 가이드

### 4-1. 지역 카테고리 추천 포스트 (핵심)
- 검색의도 예: "판교역 고기 맛집 추천", "삼성역 이자카야 BEST"
- 필수: 5~7곳 추천 + 비교표 + 상황별 요약
- 필수: 각 식당의 평점, 리뷰수, 대표메뉴, 가격대 데이터
- CTA: 각 식당 상세 페이지 링크 + AI 추천 유도
- 금지: 리뷰 원문 인용, 근거 없는 순위

### 4-2. 상황별 가이드 포스트
- 검색의도 예: "잠실 회식 어디", "판교 혼밥 맛집"
- 필수: 해당 상황 체크리스트 (인원수, 예산, 주차, 룸 등)
- 필수: 3~5곳 추천 + 각 식당이 그 상황에 맞는 이유
- CTA: AI 추천 (기분/상황 입력하면 자동 추천)

### 4-3. 지역 총정리 포스트
- 검색의도 예: "삼성역 점심 뭐 먹지", "영통역 맛집 총정리"
- 필수: 카테고리별 대표 1곳씩 + 가격대별 분류
- 필수: 지도 영역 설명 (어디에 뭐가 밀집해있는지)
- CTA: 지역 페이지 + 카테고리 페이지 내부링크

### 4-4. 비교형 포스트
- 검색의도 예: "판교 삼겹살 가격 비교"
- 필수: 동일 카테고리 3~5곳 직접 비교 테이블
- 필수: 각 항목 (가격, 평점, 메뉴, 특징) 비교
- 금지: 1곳만 압도적으로 좋다는 결론 (근거 없는 1위)

### 4-5. 시즌·트렌드 포스트
- 검색의도 예: "2026 여름 삼성역 야장"
- 필수: 시즌 연관 키워드 + 날씨 연동 추천
- 필수: 해당 시즌에만 유효한 정보 명시 (야장 운영기간 등)

## 5. 승인 실패 조건

아래 중 **하나라도** 해당하면 반려:

1. 맛집 외 주제
2. 날짜형/불릿형/이모지형 H2
3. 데이터 없는 식당 추천 (data 파일에 미존재)
4. 네이버 리뷰 원문 인용 또는 사이트 노출
5. 광고 상한 초과
6. 이미지 상한 초과
7. 내부링크가 존재하지 않는 URL
8. body 5개 이하의 얇은 글
9. H2와 바로 다음 body 첫 문장 동일
10. intro에 결론 없음
11. 비교표/추천표 없이 나열만 한 글

## 6. 신규 포스트 체크리스트

```
□ node --check posts/{id}.js 통과
□ export default post (ESM 방식)
□ data/posts.js에 메타데이터 추가됨
□ slug가 영문 키워드 기반
□ 추천 식당이 data/{region}.js에 실존
□ intro에 핵심 결론 있음
□ 비교표 또는 상황별 추천표 포함
□ H2에 날짜/불릿/이모지 없음
□ 광고 3개 이하
□ 이미지 3장 이하
□ 데이터 기준일 명시됨
□ 내부링크 앵커가 설명형
□ 네이버 리뷰 원문이 사이트에 노출되지 않음
□ body가 6개 이상
□ ending에 관련 글 3~5개 링크
□ AI 추천 서비스로 유도하는 CTA 1개 이상
□ 실제 사용자의 식당 선택에 도움이 되는가
```

## 7. 우선순위 포스팅 주제

### 1순위 — 각 지역 핵심 카테고리 (6지역 × 3~4편 = 18~24편)
- `{지역} 고기구이 맛집 추천 2026`
- `{지역} 점심 맛집 가이드 2026`
- `{지역} 이자카야·술집 추천 2026`
- `{지역} 회식 장소 추천 2026`

### 2순위 — 상황별 (각 주요 지역별 3편 = 9편)
- `{지역} 혼밥 맛집 추천`
- `{지역} 데이트 맛집 추천`
- `{지역} 가성비 점심 1만원 이하`

### 3순위 — 비교·트렌드 (5~10편)
- `삼성역 vs 판교역 직장인 점심 비교`
- `2026 여름 야장 맛집 총정리`
- `판교 테크노밸리 런치 가격대별 가이드`

## 8. 작업 유형별 세부 가이드

### 8-1. 기존 포스트 갱신
1. 대상 포스트 posts/XX.js 읽기
2. data/{region}.js에서 최신 데이터 확인 (평점, 리뷰수, 가격 변동)
3. 변경된 데이터 반영 + 기준일 업데이트
4. 누락된 식당/폐업 식당 확인
5. 상황별 추천표/비교표 정합성 확인
6. `node --check posts/XX.js`
7. 커밋 메시지 제안

### 8-2. 신규 포스트 생성
1. POST_TEMPLATE.md 구조 확인
2. data/{region}.js에서 추천 식당 실존 여부 확인
3. sections 배열 작성 (intro → image → toc → h2+body 반복 → ending)
4. 비교표/상황별 추천표 필수 포함
5. 데이터 기준일 명시
6. `node --check posts/XX.js`
7. data/posts.js 메타데이터 추가
8. 커밋 메시지 제안

### 8-3. 카테고리/지역 페이지 콘텐츠 강화
1. 해당 지역 데이터 파일에서 카테고리별 식당 수, 평균 평점, 가격대 범위 집계
2. 가이드 콘텐츠 작성 (800~1,200자)
3. 상황별 추천 (회식/데이트/혼밥) + 가격대별 가이드 포함
4. 관련 카테고리/포스트 내부링크 추가
5. `node --check` 통과 확인

### 8-4. 데이터 보강 (네이버 크롤링 후)
1. merge-naver-data.mjs로 기존 데이터와 병합
2. 스키마 확장 필드 (menuItems, keywords, naverUrl 등) 정합성 확인
3. rv 필드 사이트 노출 여부 검증 (절대 금지)
4. tags/moods/scene 자동 매핑 결과 확인
5. 전체 데이터 검증: `node --check` + `validate_name()`
