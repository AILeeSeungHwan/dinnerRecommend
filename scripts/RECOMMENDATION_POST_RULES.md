# 추천 포스팅 작성 규칙 (모범 사례: posts/10.js · 망포 가성비 점심)

> **이 문서는 dinner.ambitstock.com의 모든 추천 포스팅이 따라야 할 규칙입니다.**
> 모범 답안: `posts/10.js` (망포역 가성비 점심 1만원대 BEST 5)
> 신규 포스트·재생성·QA 시 반드시 본 규칙을 참조합니다.

---

## 1. 절대 원칙

1. **템플릿화 금지** — 모든 포스트는 식당별 unique 콘텐츠로 작성. 같은 표현 3회 이상 반복 금지.
2. **데이터 무결성** — 평점·리뷰·가격·영업시간은 데이터 그대로. 모순 표현 금지 ("평점 3.4 + 신뢰도 우위" X).
3. **솔직한 한계 명시** — 평점 < 4.0 식당 포함 시 한계 짚기 + 차별점 명시.
4. **기존 URL·발행일 절대 보호** — slug/date 변경 X.
5. **식당 원래 이름 유지** — 본문 노출 라벨도 데이터의 name과 동일하게 (예: "스타벅스 수원망포DT점" → 그대로).
6. **한국어 톤** — "~습니다" 체. 영어 노출 최소화 (식당명·약어 외).

## 2. 섹션 구조 (14개 의무)

```
1. intro    — 도입부 + 통계 카드 4개 (분석 식당·평균 평점·누적 리뷰·시작가)
2. toc      — 목차
3. ad       — 상단 광고 (slot:9463227631, format:autorelaxed)
4. h2 verdict   — 한 줄 결론 박스 (구체적 식당명 + 추천 시나리오 매칭)
5. h2 criteria  — 선정 기준 박스 (4가지 데이터 기준)
6. h2 + body (식당 카드 5곳) — 각 식당당 unique 묘사
7. ad       — 중간 광고 (slot:6297515693, format:auto)
8. h2 compare   — 5곳 한눈에 비교 표
9. h2 by-situation — 상황별 추천 카드 5개
10. h2 tips    — 방문 전 체크포인트 5개+
11. h2 faq     — FAQ 6개 (defaults open, <details open>)
12. cta        — 카테고리 전체 보기 링크
13. ending     — 운영자 노트 3 단락 + 데이터 출처 4중 신호
```

## 3. 식당 카드 의무 요소 (5곳 동일 적용)

각 식당 카드는 다음 8개 요소를 모두 포함:

### 3-1. 이미지 (최우선)

- **헤더 이미지** 1장: `<img src="..." referrerpolicy="no-referrer">`
  - 1순위: 데이터의 `imageUrl` (네이버 플레이스)
  - 2순위: 카카오 이미지 검색 API (Kakao dapi)
  - 3순위: 카테고리 그라데이션 fallback (배경색 + 이모지)
- **갤러리 3장**: imageUrl2/3/4 또는 카카오 검색 결과
- 이미지 없으면 카테고리 fallback 그라데이션 (CAT_GRAD 매트릭스):
  - 한식: #F97316~#DC2626 + 🍚
  - 국밥: #EA580C~#B45309 + 🍲
  - 고기: #B91C1C~#7F1D1D + 🥩
  - 중식: #DC2626~#F59E0B + 🥢
  - 일식: #0EA5E9~#1E40AF + 🍣
  - 이자카야: #4338CA~#1E1B4B + 🍶
  - 양식: #C026D3~#7C3AED + 🍝
  - 카페: #92400E~#451A03 + ☕
  - 치킨: #F59E0B~#DC2626 + 🍗

### 3-2. 헤더 뱃지 5개 (가로 배열)

```
🏆 순위 라벨 (가성비 1순위 / 검증 우선 / 회식·모임 1순위 등)
★ 평점 · 리뷰 N건
💰 가격대 lo~hi원
🚗 주차 가능 / 🌙 심야 영업 / 📷 사진 명소 / ✅ 예약 가능 (특성별)
✓ 인허가 검증 (행정안전부)
```

### 3-3. 차별화 묘사 (200~300자)

식당별 진짜 강점 1~2가지 자율 선정:
- 가격 위치 (저렴·평균·고가 vs 카테고리 평균)
- 검증 신뢰 (리뷰 수 표본 크기)
- 평점 우위 (카테고리 평균 대비)
- 운영 안정성 (운영 기간·인허가)
- 영업 차별화 (이른 시간·심야·24시간·DT)
- 메뉴 차별화 (시그니처·단일 회전·세트)

**금지 표현 (반복 회피)**:
- "신뢰도 측면에서 우위에 있습니다"
- "전용 주차장이 없는 편이라 대중교통 이용을 권장합니다"
- "평가가 카테고리 평균 수준이라"
- "대표 메뉴 정보는 매장에 직접 확인을 권장드립니다"
- "줄 세운 결과"

### 3-4. 후기 키워드 정리 박스 (직접 인용 금지)

**나쁜 예**: "저녁으로 먹으러 왔는데, 양도 많고 맛있어요" — 직접 인용 X
**좋은 예**: '"양이 많다 · 맛있다 · 분위기 깔끔하다" 같은 평이 자주 언급되었습니다.'

박스 스타일: `background:#FFF7E0; border-left:3px solid #F59E0B; padding:12px 16px; border-radius:0 6px 6px 0`
- 제목: `📝 방문자 후기 키워드 정리`
- 내용: 키워드 3개 · 문장 1줄로 요약

데이터 소스: `data/reviews_summary.json` → `themes` 키워드 활용 (직접 quote 인용은 금지)

### 3-5. 메뉴 그리드 (3~6개)

`grid-template-columns:repeat(3,1fr); gap:8px` 패턴.
각 메뉴 카드:
- 메뉴명 (굵게)
- 가격 (또는 "매장 문의")
- 라벨: **최저가** / **시그니처** / **인기** / **가성비** (해당 시)

메뉴 데이터 없으면: 카테고리 표준 메뉴를 추정 가격대로 (예: 신선설농탕은 체인 표준 가격으로 작성).

### 3-6. 추천 시나리오 태그 4개

`background:#EEEDFE; color:#26215C; padding:5px 11px; border-radius:6px`
예: 혼밥 가능 · 점심 7,500원 · 주차 가능 · 깔끔한 곳

### 3-7. 정보 라인 (📍🕐🚗📞)

```
📍 위치 [주소] · 🕐 영업 [시간] · 🚗 주차 [상태] · 📞 전화 [번호]
```

영업시간 데이터 검증:
- "11:00 AM~12:00 PM" 같은 1시간 영업 → "영업일 매장 확인"으로 정규화
- "휴무일" 단독 표시 → "영업일 매장 확인"

### 3-8. 상세 정보 버튼 (필수 디자인)

```html
<div style="text-align:center">
  <a href="/{region}/restaurant/{name}" style="
    display:inline-block;
    margin-top:16px;
    padding:12px 22px;
    background:#FCD34D;
    color:#111827;
    border-radius:10px;
    text-decoration:none;
    font-size:14px;
    font-weight:900;
    border:2px solid #111827;
    box-shadow:0 3px 0 #111827;
  ">👉 {식당명} 상세 정보 보기 →</a>
</div>
```

색상·테두리·그림자 임의 변경 금지.

## 5. 운영자 노트 (ending 섹션)

3 단락 구조 의무:

1. **데이터 필터링 결과**: "{region} {N}곳에서 {카테고리} 필터링 → {N}곳"
2. **솔직한 한계 인정**: 평점 낮은 식당 한계 + 차별점
3. **사용자 가이드**: 평점 우선 vs 표본 우선 vs 가성비 우선

스타일:
- 배경 `#EEEDFE` (보라 톤)
- 큐레이터 아바타 "큐" (M 아님)
- "오늘뭐먹지 큐레이터" 표시

## 6. 데이터 출처 (4중 신호)

ending 섹션 하단 박스:
- 행정안전부 전국일반음식점표준데이터 (data.go.kr)
- 한국관광공사 국문 관광정보 (api.visitkorea.or.kr)
- 식품의약품안전처 위생등급 (data.mfds.go.kr)
- 네이버 플레이스 + 다음 검색 종합

기준일 표기 의무: "YYYY년 MM월 DD일 기준 정보입니다."

## 7. 14개 자동 QA 체크리스트

QA 에이전트가 다음 14개를 검증:

1. 본문 분량 5,000자 이상 (이미지 태그 제외)
2. 한 줄 결론 박스에 구체적 식당명 3개 이상
3. 같은 표현 3회 이상 반복 없음 (금지 표현 목록 확인)
4. 평점·리뷰 모순 없음 (평점 < 4.0 + "검증 우위" 동시 표기 X)
5. 영업시간 1시간 영업·"휴무일" 단독 표시 0건
6. 카테고리 매칭 100% (제목 카테고리와 본문 식당 카테고리 일치)
7. 식당당 메뉴 그리드 3개+
8. 시그니처 메뉴 명시 (각 식당당 1개+)
9. 외부 출처 1~2개 (행정안전부·관광공사·식약처 중)
10. FAQ 6개+ (모두 `<details open>` 사용)
11. 운영자 노트 3 단락 (필터링 + 한계 + 가이드)
12. AI 정형 종결어 (총정리·살펴봅니다·분석합니다) 사용 0~1회
13. "~습니다" 체 80%+
14. 핫링킹 이미지 안전 처리 (`referrerpolicy="no-referrer"`)

**임계점**: 12개 이상 통과 → 자동 발행. 11개 이하 → 재작성.
